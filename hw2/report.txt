1. Browser Cache Hit (브라우저 캐시): "보이지 않는 것이 증거"
Wireshark는 네트워크 선을 타고 오가는 데이터만 잡습니다. 만약 브라우저가 "어? 이 페이지(example.com)는 아까 접속했으니까 내 컴퓨터(캐시)에 저장된 걸 보여줘야지"라고 판단하면, 서버로 요청을 아예 보내지 않습니다.

2. Server Cache Miss (서버 캐시): "404 Not Found"

설명: 서버가 "찾아봤는데 없더라"라고 응답했습니다. 만약 CDN이나 프록시 서버(예: Varnish, Nginx 등)를 거쳤다면 HTTP 헤더(Wireshark 하단 패킷 상세 창)에 X-Cache: MISS 같은 문구가 있을 수 있습니다. 하지만 현재 CSV 파일(리스트 뷰)에서는 헤더 내부까지는 안 보이고 결과(404)만 확인됩니다.

보고서 작성 팁: Wireshark 원본 화면에서 Packet 1102를 클릭하고 아래 Hypertext Transfer Protocol 부분을 열어보세요. Via, X-Cache, Age 같은 헤더가 있다면 서버 측 캐시 상태를 더 정확히 알 수 있습니다.
